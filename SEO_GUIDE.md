## 📋 Step-by-Step: Submit to Google Search Console

### **Step 1: Create a Google Search Console Account**
1. Go to: https://search.google.com/search-console
2. Sign in with your Google account (create one if needed)
3. Click the "+" button to add a property

### **Step 2: Verify Domain Ownership**
Choose one of these verification methods:

#### **Method 1: HTML File Upload (Recommended for GitHub Pages)**
1. In Search Console, select "URL prefix" property type
2. Enter: `https://omarabualrub1.github.io/cocanillaa/`
3. Choose "HTML file" verification method
4. Download the verification HTML file (e.g., `google******.html`)
5. Upload this file to your project root folder in the test directory
6. Commit and push to GitHub
7. Click "Verify" in Search Console

#### **Method 2: Meta Tag**
1. Copy the meta tag provided by Search Console
2. Add it to your `<head>` section (already partially prepared in your HTML)
3. Commit and push changes
4. Click "Verify" in Search Console

#### **Method 3: Google Analytics/Google Tag Manager**
If you have these linked to your GitHub Pages account:
1. Search Console can verify automatically
2. No additional steps needed

### **Step 3: Submit Your Sitemap**
1. In Search Console, go to "Sitemaps" section
2. Click "Add/test sitemap"
3. Enter: `sitemap.xml`
4. Click "Submit"
5. Search Console will crawl and index your pages

### **Step 4: Request Indexing**
1. Go to "URL Inspection" tab
2. Enter your website URL: `https://omarabualrub1.github.io/cocanillaa/`
3. Click "Request Indexing"
4. Do this for key pages:
   - `/` (home)
   - `/#products`
   - `/#contact`
